## What It Does

```
Sources             Pipeline                    Outputs
──────────          ──────────────              ──────────────
Hacker News  ─┐                                 ┌─ Markdown (EN/ZH)
RSS / Atom   ─┤     Route      Score     Enrich │
Reddit       ─┼──▶  to     ──▶ 0-10  ──▶ with  ─┼─ HTML Report
Telegram     ─┤     Groups     & Filter  BG     │
GitHub       ─┘                                  ├─ WxPusher (Mobile)
                                                 ├─ AI Podcast (MP3)
                                                 └─ GitHub Pages
```

1. **Fetch** — Pull content from all sources concurrently
2. **Deduplicate** — Merge identical URLs across platforms
3. **Route** — Assign items to configurable groups (headlines, watched repos, trending, etc.)
4. **Score** — AI rates each item 0-10 for technical depth, novelty, and impact
5. **Filter** — Keep items above your threshold; bypass scoring for specific groups
6. **Enrich** — Web-search for background context, collect community discussions
7. **Classify** — AI clusters items into sub-topics within each group
8. **Render** — Generate outputs: Markdown, newspaper-style HTML, mobile brief, and podcast

## Features

### Core

- **Multi-Source Aggregation** — Hacker News, RSS/Atom, Reddit, Telegram, GitHub (releases & user events)
- **AI Scoring** — Works with any OpenAI-compatible API (Qwen, GPT-4, Claude, Gemini, DeepSeek, etc.)
- **Bilingual Output** — Full reports in both English and Chinese
- **Content Enrichment** — Background knowledge and community discussions for every item
- **Cross-Source Dedup** — Automatic URL-based and topic-based duplicate merging

### Group System

Items are routed into configurable groups, each with its own scoring prompt, threshold, and enrichment mode:

```jsonc
"groups": [
  {
    "name": "Headlines",
    "categories": ["hackernews", "tech-analysis", "ai-news-zh"],
    "scoring": { "enabled": true, "threshold": 7.0 }
  },
  {
    "name": "Watched Repos",
    "categories": ["github-updates"],
    "scoring": { "enabled": false }  // bypass — always included
  }
]
```

### Multi-Channel Output

| Channel | Description |
|---------|-------------|
| **Markdown** | Full-length ZH/EN summaries with TOC, scores, tags, references |
| **HTML Report** | Newspaper-style multi-page layout with sidebar navigation and sub-topic grouping |
| **WxPusher** | Mobile push notification via WeChat with complete summaries |
| **AI Podcast** | Two-host dialogue script generated by LLM, synthesized via CosyVoice TTS, delivered as RSS |
| **GitHub Pages** | Static site auto-deployed from `docs/` — summaries, HTML reports, and podcast RSS |
| **Email** | Self-hosted SMTP/IMAP newsletter with automatic subscribe/unsubscribe |

### AI Podcast

Horizon can generate a daily podcast episode from the top news:

```
Top items → LLM dialogue script → CosyVoice TTS → FFmpeg merge → MP3
          → Upload to GitHub Release → Update RSS feed → Push to AntennaPod
```

- Two distinct AI hosts with natural conversation flow
- DashScope CosyVoice for high-quality Chinese speech synthesis
- Standard Podcast RSS 2.0 feed, compatible with any podcast app

### MCP Integration

Built-in [MCP](https://modelcontextprotocol.io/) server lets AI assistants drive the pipeline:

```bash
uv run horizon-mcp
```

Pipeline tools: `hz_validate_config`, `hz_fetch_items`, `hz_score_items`, `hz_filter_items`, `hz_enrich_items`, `hz_generate_summary`, `hz_run_pipeline`.
Run history: `hz_list_runs`, `hz_get_run_meta`, `hz_get_run_stage`, `hz_get_run_summary`, `hz_get_metrics`.

## Quick Start

### 1. Install

```bash
git clone https://github.com/Ming-321/horizon.git
cd horizon
uv sync          # or: pip install -e .
```

### 2. Configure

```bash
cp .env.example .env
cp data/config.example.json data/config.json
```

Edit `.env` with your API keys, then customize `data/config.json`. See the [Configuration Guide](https://ming-321.github.io/horizon/configuration) for the full reference.

### 3. Run

```bash
uv run horizon               # default 24h window
uv run horizon --hours 48    # last 48 hours
uv run horizon --from-cache  # re-render from cached data (no API calls)
```

Reports are saved to `data/summaries/`, HTML to `data/html/`, and podcasts to `data/podcasts/`.

Preview HTML reports locally:

```bash
uv run horizon-server            # serves data/html/ at http://localhost:8080
uv run horizon-server --port 3000
```

### 4. Automate (Optional)

Use a cron job or GitHub Actions to run daily:

```bash
# crontab -e
0 7 * * * cd /path/to/horizon && uv run horizon --hours 24
```

## Supported Sources

| Source | What It Fetches | Comments |
|--------|----------------|----------|
| **Hacker News** | Top stories by score | Top N comments |
| **RSS / Atom** | Any feed URL | Supports GitHub Trending, Reddit RSS, blog feeds |
| **Reddit** | Subreddits + user posts | Top N comments |
| **Telegram** | Public channel messages | Web preview scraping (no API key needed) |
| **GitHub** | User events & repo releases | Commit aggregation for watched repos |

## Project Structure

```
src/
  ai/          — AI client, analyzer, summarizer, prompts
  scrapers/    — RSS, GitHub, HN, Telegram scrapers
  renderers/   — Brief, HTML, topic classifier
  services/    — Email, WxPusher, podcast pipeline
  mcp/         — MCP server integration
  storage/     — Run history, metrics, cache
data/          — config.json, prompts, generated outputs
tests/         — pytest test suite
docs/          — GitHub Pages site + planning docs
```

## Configuration Overview

```jsonc
{
  "ai": {
    "provider": "openai",
    "model": "qwen3.5-plus",
    "base_url": "https://dashscope-intl.aliyuncs.com/compatible-mode/v1",
    "languages": ["zh", "en"]
  },
  "sources": { /* HN, RSS, Reddit, Telegram, GitHub */ },
  "groups": [ /* route items by category, per-group scoring/enrichment */ ],
  "output": {
    "brief": { "enabled": true, "top_n": 10 },
    "html": { "enabled": true },
    "podcast": { "enabled": true, "tts_model": "cosyvoice-v3-flash" }
  },
  "notifications": {
    "wxpusher": { "enabled": true }
  }
}
```
